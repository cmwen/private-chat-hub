## Design Principles (Updated for v2)

### Core Philosophy

1. **Transparency**: Users see what tools/models are being used, in real-time
2. **Power Without Overwhelm**: Progressive disclosure for advanced features
3. **Local-First**: Reinforce data ownership and privacy throughout
4. **Task-Oriented**: Support multi-step workflows, not just chats
5. **Accessibility**: Support for TTS, voice, visual/motor abilities

### What Stays the Same

- âœ… Material Design 3 foundation
- âœ… Dark mode first approach

### What's New in v2

- âœ¨ Native share/TTS controls
